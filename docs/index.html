Chameleon Theme Engine

Project Overview

A flexible, rule-based, multi-tenant theme engine for Flutter apps...

Key Features

  • Declarative Onboarding
  • Rule-Based Runtime Switching
  • Strict Widget Isolation
  • Robust Per-Token Fallback
  • Dynamic Custom Extensions
  • Persisted State

Onboarding a New Tenant

To configure a new tenant (e.g., mybrand):

1. Add Tenant Config JSON: Create assets/configs/tenants/mybrand.json...

{
  "themes": {
    "default": {
      "colors": {
        "primary": "#005BAC",
        "secondary": "#4A5568",
        "surface": "#FAFAFA",
        "onPrimary": "#FFFFFF"
      },
      "typography": {
        "fontFamily": "Inter",
        "fontWeight": 400.0,
        "fontSize": 14.0
      },
      "radii": {
        "card": 12.0,
        "button": 8.0
      },
      "spacing": {
        "xs": 4.0,
        "sm": 8.0,
        "md": 16.0
      },
      "semanticMap": {
        "buttonBackground": "primary",
        "buttonForeground": "onPrimary",
        "cardSurface": "surface",
        "cardRadius": "card",
        "buttonRadius": "button"
      }
    }
  },
  "themeRules": {
    "default": "default"
  }
}

2. Register Asset: Make sure the assets directory is declared in the root pubspec.yaml:

flutter:
  assets:
    - assets/configs/tenants/ # Register tenant folder

Usage Guidelines

1. Bootstrapping (Dependency Injection): Initialize the theme engine at application startup in your main() method:

2. Wrapping your App: Expose the ThemeService through a ListenableBuilder to trigger rebuilds on variant changes:

3. Consuming Semantic Roles in Widgets: To prevent coupling widgets to raw values, never hardcode colors. Import the theme engine package and consume named semantic roles via context.appTheme:

import 'package:chameleon_theme/chameleon_theme.dart';

Widget build(BuildContext context) {
  final theme = context.appTheme;

  return Container(
    padding: EdgeInsets.all(theme.cardRadius),
    decoration: BoxDecoration(
      color: theme.cardSurface,
      borderRadius: BorderRadius.circular(theme.cardRadius),
      border: BorderSide(color: theme.inputBorder),
    ),
    child: ElevatedButton(
      style: ElevatedButton.styleFrom(
        backgroundColor: theme.buttonBackground,
        foregroundColor: theme.buttonForeground,
        shape: RoundedRectangleBorder(
          borderRadius: BorderRadius.circular(theme.buttonRadius),
        ),
      ),
      onPressed: () {},
      child: Text(
        'Submit',
        style: TextStyle(
          fontFamily: theme.fontFamily,
          fontSize: theme.fontSize,
        ),
      ),
    ),
  );
}

Development Workflow

This section outlines the common terminal commands and contribution policies...

fvm flutter pub get
Install Flutter package dependencies.
fvm dart run build_runner build --delete-conflicting-outputs
Regenerate JSON serialization code.
fvm flutter test
Run all tests.
fvm flutter analyze
Run static code analysis.
cd example && fvm flutter run
Run the showcase interactive app.

Pre-Commit Checks

  • No ! operators in Dart code.
  • No print calls.
  • No raw Color literals outside theme library.
  • Static analysis must pass with zero errors.
  • All tests must pass.

Prohibited Practices & Enforcement

  • No Force Unwraps (!): Sound null safety must be maintained...
  • No Raw print(): Use dart:developer log()...
  • No Hardcoded Colors: Widgets must only consume semantic roles...
